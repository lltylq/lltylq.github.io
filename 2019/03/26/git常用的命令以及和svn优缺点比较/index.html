git常用的命令以及和svn优缺点比较

发表于 2019-03-26 | 更新于 2019-06-09 | 阅读次数：
本文字数： | 阅读时长 ≈

git来历（牛人作品）：Linux系统开源后，全世界的码农都开始了开发，10年后代码已经无法管理（linux反对使用svn，应为集中式管理速度太慢了），linus自己发了2周的时间开发了git分布式管理系统一直火到了现在。

  1. 系统特点
    svn是集中式管理系统，版本库是存放在中央管理器中的，我们工作的时候需要在中央管理器中取的最新的版本库然后再开始工作，干完活后又必须把他提交到中央管理器上。
    git是一个分布式管理系统，什么叫分布式管理系统呢？就是说它是没有中央管理器，每个人的电脑上都是一个完整的版本库，当你写完了代码就可以将自己的修改的地方推送给对方，就能相互看到修改了（比如今天你的同事生病了，他的电脑没有在公司，因此分布式管理系统也需要一个充当“中央管理器”电脑，但是这个服务器作用只是方便大家修改代码的交换。）。

  2. 安全性
    svn使用的集中式管理一旦那台服务器出现故障，无法在次交互，很影响团队之间的协作，但是git就不一样了， 他可以任意的建立分支，交互十分简单。

  3. 分支方面
    svn的分支使用十分麻烦，关键的问题是速度很慢（拉分支时间慢，相当于copy），并且一个人新建了一个分支，团队的所有的人都会新建一个分支，这样是任务变得很复杂，但是git分支却完美的解决了这个问题，git使用git branch 分支名 新建分支后是直接在原版本库上延伸后新加一个指针，这样不会影响到其他人，删除个人分支就直接删除指针就ok啦，非常方便。

git常见的命令

Workspace：工作区
Index / Stage：暂存区
Repository：仓库区（或本地仓库）---通常使用fork 讲远程仓库里面新建自己的一个个人本地分支
Remote：远程仓库
fork、barnch、clone的区别

代码提交 

# 本地的文件提交到暂存区
$ git add .

# 提交暂存区到仓库区
$ git commit -m [message]

# 提交暂存区的指定文件到仓库区
$ git commit [file1] [file2] ... -m [message]

# 提交工作区自上次commit之后的变化，直接到仓库区
$ git commit -a

# 提交时显示所有diff信息
$ git commit -v

# 使用一次新的commit，替代上一次提交
# 如果代码没有任何新变化，则用来改写上一次commit的提交信息
$ git commit --amend -m [message]

# 重做上一次commit，并包括指定文件的新变化
$ git commit --amend [file1] [file2] ...

分支命令

#查看本地的分支
$git branch
#新建一个本地为dev的分支，不切换。
$git branch -dev
#新建一个本地为dev的分支，且切切换到dev上。
$git checkout-v dev
#新建一个本地为dev的分支，不切换。
$git branch -dev
# 建立追踪关系，在现有分支与指定的远程分支之间
$ git branch --set-upstream [branch] [remote-branch]

查看命令

#查看本地修改的文件
$git status  
#查看本地和暂存区文件的区别
$git diff
